# ADR-0002: 서비스 경계(Service Boundary) 결정

- 상태: 승인(Approved)
- 날짜: 2026-03-08
- 작성자: Codex

## 배경
- `runtime-poc` 결과에서 단말 로컬 런타임 기반 실행은 `codex` 실행체 부재로 안정적인 운영 기준을 충족하지 못했다.
- 앱/서버 역할과 책임(세션, 파일 접근, 모델 설정)의 소유 경계를 명확히 정의할 필요가 있다.

## 결정
**B안(서버 오프로딩 중심, 앱은 API 클라이언트)을 채택한다.**

- 앱은 사용자 인터페이스, 입력 수집, 스트리밍 출력 렌더링, 인증 토큰 관리에 집중한다.
- Codex 실행, 세션 상태 관리, 파일시스템 접근 제어, 모델 파라미터 적용은 서버에서 수행한다.

## 경계 정의 표

| 영역 | 앱(클라이언트) 책임 | 서버 책임 | 비고 |
|---|---|---|---|
| 세션 수명주기(생성/복구/종료) | 세션 생성 요청, 재연결 트리거, 사용자 종료 요청 UI 제공 | 세션 ID 발급, 상태 저장/복구, 종료 처리(정리/만료) | 세션의 authoritative state는 서버가 보유 |
| 파일 접근 범위 | 로컬 파일 선택/업로드, 다운로드 결과 저장 | 작업공간 마운트/샌드박스, 접근 정책 적용, 실행 중 파일 I/O | 앱은 서버 파일시스템에 직접 접근하지 않음 |
| 모델 설정 소유권 | 사용자 의도 입력(예: 모델 선호도) 및 표시 | 실제 모델/파라미터 결정, 정책 기반 강제/검증, 감사 로그 | 운영 일관성을 위해 최종 설정 권한은 서버 |

## 근거
1. 단말 환경 편차(ABI/런타임 구성)에 따른 실행 실패 리스크를 서버 단일 런타임으로 흡수할 수 있다.
2. 보안·감사 관점에서 파일 접근 및 모델 설정을 서버에서 일원화하면 통제 가능성이 높다.
3. 세션 복구/장애 전파 제어를 서버에서 일괄 처리하면 운영 복잡도를 줄일 수 있다.

## 이번 분기 제외 항목(Out-of-Scope)
B안 채택 기준으로 이번 분기에는 아래를 범위에서 제외한다.

1. **앱 내 로컬 Codex 실행 기능**(온디바이스 런타임 포함)
2. **오프라인 단독 실행 모드**(서버 연결 없이 작업 지속)
3. **앱의 서버 작업공간 직접 마운트/직접 편집 기능**
4. **사용자 단말별 임의 모델 바이너리/엔진 플러그인 로딩**
5. **멀티 리전 액티브-액티브 세션 동기화**(단일 리전 고가용성 우선)

## 결과 및 후속 조치
- 앱/서버 API 스펙은 “클라이언트 요청 + 서버 authoritative state” 원칙으로 업데이트한다.
- 세션 복구 시나리오(네트워크 단절/앱 재시작/서버 워커 재기동) E2E 테스트를 우선순위로 반영한다.
- 모델 설정 변경은 서버 정책 레이어를 통하도록 구현한다.
